使用Hexo的Next主题搭建个人github博客

发表于 2019-01-01 分类于 建站 阅读次数： 本文字数： 5.9k 阅读时长 ≈ 5 分钟

摘要

本文主要介绍了如何使用Hexo搭建属于自己的github博客，以及更换站点主题为Next；Hexo常见配置；Next 主题常见配置等等；如果觉得功能还不够强大，还可以参考文档里的链接，再根据自己的需求再进行配置；

说明

首先在github上新建两个代码仓库，例如blog和blog-origin两个代码库，其中blog代码库是最终站点的代码，blog-origin是基于hexo主题的源代码（当然也可以只建一个代码库，不过会比较混乱，在提交时也会有问题，因此还是建议分开两个代码库）

说明：github代码仓库的安装假装大家都会了，如有需要再补充~

安装

blog-origin代码库

补充说明：

  • 每次发布代码需要执行hexo g 和 hexo d 两个命令，比较麻烦，可以修改 package.json 文件增加自定义指令，例如修改 “deploy”: “hexo g && hexo d”，这样修改完代码发布的时候只需要执行 npm run deploy 即可，可根据个人习惯自行修改设置；

  • 如果推送完代码到github上没有生效，请强制刷新页面；

blog代码库

  • 代码：blog的代码是由blog-origin编译得来，通过hexo d的方式推送到github代码库中，不用手动处理~

  • 设置：需要在 Settings 里 GitHub Pages里选择对应的分支就会得到默认的站点地址~

主题外观设置

next主题一共支持4种外观，在themes/next/_config.yml配置文件种搜索scheme字段，4种样式风格如下：

  • Muse - 默认 Scheme，这是 NexT 最初的版本，黑白主调，大量留白
  • Mist - Muse 的紧凑版本，整洁有序的单栏外观
  • Pisces - 双栏 Scheme，小家碧玉似的清新
  • Gemini - 紧凑集中版双栏 Scheme，本网站采用的主题

使用FontAwesome5

Next主题集成的是4.6.2版本的fontawesome，如果要升级到5.X版本，一般是去修改主题配置文件_config.yml中的fontawesome字段，但是这个修改会导致展示不出来icon，因此直接去修改页面头文件的模板，找到font_awesome_uri，将其设置为https://use.fontawesome.com/releases/v5.13.0/css/all.css即可。

自定义设置

code颜色更改

个性化的设置部分已经包括了大部分常用的功能，如果还需要更多细节上的修改，则需要修改源码，例如调整本博客调整了code部分的颜色和背景色，需要修改themes/next/source/css/_variables/base.styl文件，查找$code-foreground和$code-background，分别设置字体颜色以及背景色即可。
